# Self Review

## Current Project Status

This project is a local-first lecture knowledge site generator with a local FastAPI dashboard. It reads lecture materials from `input_materials/`, generates versioned static sites in `output_sites/`, and writes metadata plus QA reports for each run.

Confirmed supported inputs are `.pdf`, `.md`, `.markdown`, and `.docx`. `requirements.txt` also includes `python-pptx`, but the current generator path does not list PPTX as a supported input in `scripts/utils.py`.

The system has two explicit modes:

- Offline Mode: local source explorer with cleaned excerpts, source references, search, slide figures/gallery, lecture map, cross-lecture references, basic glossary, template practice, versioned output, and QA.
- Online LLM Mode: optional provider-backed generation for Chinese structured summaries and advanced/creative practice paths. LLM output must be JSON-schema validated before rendering.

## How to Judge a Generated Version

A generated version is acceptable when:

- `metadata.json` correctly records generation mode and LLM telemetry;
- `qa_report.json` status is `passed`, or `warnings` are explicitly understood and acceptable for that run;
- ordinary body paragraphs have no metadata leakage, long English slide dumps, raw PDF dumps, or raw LLM Markdown;
- Source-grounded Notes use validated Chinese structured summary fields in Online LLM Mode, or cleaned excerpts/source references in Offline/fallback mode;
- source quotes are short and not excessive;
- sources are rendered in collapsible panels;
- slide figures, captions, and page references remain available;
- bilingual mode banner and feature matrix still work if UI text was touched;
- Practice Generator respects selected question type;
- generated output does not contain API keys or `.env` values.
